#三次握手 
客户端与服务器进行http发生返回过程中，需要创建一个TCP connection HTTP只有请求和响应的概念，不存在连接。TCP里创建一个二者的通道。可存在多个http请求，可以通过声明让其一直保持tcp连接

两个序号 三个标志位

seq 序号   用来表示向服务器发送的字节流
ack 确认序号   ACK =1时，ack才有效，ack = seq+1；
标志位
ACK：确认信号有效
SYN：发起一个新连接
FIN：释放一个新连接

client与server
第一次发送：客户端发起SYN=1，Seq =x;
第二次：服务器端接收并发送 SYN =1,ACK =1 ，seq = y,ack = x+1; 使用了ack对Ade数据包进行确认   收到了seq = x的数据包，准备接收x+1的数据包，于是ack =x+1，
服务器告诉客户端自己的数据包为y
客户端接收并发送  ACK = 1，ack = y+1; seq = x+1；
